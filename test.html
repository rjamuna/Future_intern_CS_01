Web Application Security Report

Performed By: Jamuna

Date: July 22, 2025

Tool Used: OWASP ZAP

Target: http://testphp.vulnweb.com/

1. Summary

This report summarizes the findings from an automated security test conducted using OWASP ZAP on a vulnerable test application. Multiple vulnerabilities of varying severity were identified.

2. Findings and Risk Ratings

  • High: User Controllable HTML Attribute (Potential XSS)
  • Medium: Absence of Anti-CSRF Tokens
  • Medium: Missing CSP Header
  • Medium: Missing Anti-clickjacking Header
  • Low: X-Powered-By Header Leak
  • Low: Server Header Leak
  • Low: Missing X-Content-Type-Options
  • Info: Authentication Form Detected
  • Info: Charset Mismatch
  • Info: Modern Web Application Detected

3. Recommendations

  • Implement CSRF tokens on all forms.
  • Set Content-Security-Policy and X-Frame-Options headers.
  • Remove or mask X-Powered-By and Server headers.
  • Sanitize and validate all user input to prevent XSS.
  • Enforce consistent character encodings.
  • Use HTTPS and strong password policies.

4. Conclusion

Jamuna successfully conducted a ZAP security scan revealing critical security gaps including potential XSS vulnerabilities, missing headers, and exposure of server information. Mitigation of these issues will greatly improve the security posture of the tested application.
